## 1. 产品概述

「安心号」是一款面向中老年家长的适老化移动应用，帮助家长代替未成年子女处理游戏账号交易（卖号退坑/买号入坑）。核心解决家长不熟悉游戏账号交易规则、容易被催促转账、泄露验证码或误信陌生人的痛点，突出"看得懂、敢操作、出问题找得到人"的产品理念。

- 目标用户：45-65岁中老年家长，替未成年子女处理退坑账号或帮外出上学的孩子完成交易
- 核心价值：用大白话讲清担保流程、用人性化设计降低操作门槛、用多重保障消除风险隐患

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 家长（主账号） | 手机号注册 | 发起交易、查看订单、联系客服、邀请家属协同 |
| 家属（协同账号） | 家长邀请链接 | 仅查看订单进度和交易凭证，不可操作 |

### 2.2 功能模块

1. **首页**：交易流程大字版说明、交易术语翻译、常见骗局语音提醒入口、平台担保收款说明、快捷操作入口
2. **安心卖号**：一步一步资料填写、账号价值粗估、换绑操作电话辅助、卖号流程进度指引
3. **安心买号**：选购流程引导、验号结果可视化展示、风险提示、买号流程进度指引
4. **订单进度**：订单全流程追踪、超时节点自动提醒、家属协同查看、交易凭证自动归档
5. **客服帮助**：人工客服一键直连、换绑操作电话辅助、常见问题FAQ、交易纠纷申诉

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|

## 3. 核心流程

### 3.1 卖号流程

家长打开App → 点击"我要卖号" → 选择游戏 → 分步填写账号信息 → 系统粗估价格 → 确认发布 → 买家下单 → 平台验号 → 双方确认 → 买家付款至平台 → 卖家换绑 → 平台放款给卖家 → 交易完成 → 凭证归档

```mermaid
flowchart TD
    "A[家长打开App]" --> "B[点击我要卖号]"
    "B" --> "C[选择游戏类型]"
    "C" --> "D[分步填写账号信息]"
    "D" --> "E[系统粗估价格区间]"
    "E" --> "F[确认发布卖号信息]"
    "F" --> "G[买家下单]"
    "G" --> "H[平台验号]"
    "H" --> "I[双方确认交易]"
    "I" --> "J[买家付款至平台担保]"
    "J" --> "K[卖家换绑账号]"
    "K" --> "L[平台放款给卖家]"
    "L" --> "M[交易完成凭证归档]"
```

### 3.2 买号流程

家长打开App → 点击"我要买号" → 选择游戏筛选账号 → 查看账号详情 → 查看验号结果 → 确认购买 → 付款至平台担保 → 卖家换绑 → 验收确认 → 平台放款 → 交易完成 → 凭证归档

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：暖橙色(#F97316)传达温暖与安全感，辅以深棕色(#7C3A0E)作文字色
- **辅助色**：浅米色(#FFF7ED)背景、绿色(#22C55E)表示安全、黄色(#EAB308)表示注意、红色(#EF4444)表示风险
- **按钮风格**：大圆角(16px)按钮，最小点击区域48x48px，带轻微阴影增加可点击感
- **字体**：标题20-24px加粗、正文18px常规、辅助文字16px，行间距1.6以上
- **布局风格**：卡片式布局，大间距(20px+)，底部Tab导航，顶部固定标题栏
- **图标风格**：线条粗(2.5px)的圆角图标，配文字标签，不依赖纯图标传达含义
- **整体风格**：温暖、可信、朴素，像"热心的邻居阿姨在帮你办事"的感觉

### 4.3 响应式设计

- **移动优先**：本产品为移动App，设计基准375px宽度，适配至428px
- **触控优化**：所有可点击元素最小48x48px，按钮间距≥12px，防止误触
- **大字模式**：支持系统级字体缩放，最大支持1.5倍放大不破版
- **横屏适配**：主要页面支持横屏浏览，布局自动调整

### 4.4 无障碍设计

- 所有文字与背景对比度≥4.5:1（WCAG AA级）
- 语音播报功能覆盖所有关键信息和提示
- 不依赖颜色单独传达信息，同时使用图标/文字辅助
- 支持屏幕阅读器
